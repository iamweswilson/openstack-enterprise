VMWare with OpenStack

4 ways to architect OpenStack on VMWare

1Multi-Hypervisor Environment

Running Both vSphere and Alternate Hypervisors

In this scenario, OpenStack is used as the control plane to manage a multi-hypervisor cloud, running both vSphere and alternate hypervisors such as KVM or Xen. This approach consolidates cloud management while allowing applications to be hosted on the environment best suited for them, maintaining a single point of control and API access.

Intel has been running this architecture with vSphere and OpenStack since 2012, with 1,500 VMs across two datacenters. With the combined environment “overseen” with OpenStack controllers, they can leverage different hypervisors while keeping operations and user experience consistent. The cloud environment enables the company’s developers to add value to the company with new applications built using the OpenStack APIs.

2Commercial Distributions

VMware Technologies Through Commercial OpenStack distributions

These OpenStack distributions from joint OpenStack ecosystem and VMware partnerships are enabled to support the VMware ESXi hypervisor, in addition to others, to easily deploy OpenStack cloud. Each offers value-added services and support that complement and enhance OpenStack to meet differing enterprise needs.

Wells Fargo is successfully using VMware technologies in its OpenStack deployment using the Mirantis Fuel distribution. The company has been leveraging vSphere for customer instances of xxxxx. With HA and vMotion enabled, Wells Fargo has experienced an extra level of built-in resiliency.

3VIO

VMWare Integrated OpenStack

VMware Integrated OpenStack (VIO) offers VMware users an alternative that combines the benefits of OpenStack with those of VMware in a single stack. This architecture uses VMware’s Software Defined Datacenter infrastructure products such as ESXi, VSAN, NSX and vCenter. As of the publication of this paper, it is new – generally available on March 13, 2015.

VIO can make it easier for IT to run a production-grade OpenStack deployment on top of their existing VMware infrastructure , gaining new cloud capabilities with the VMware interfaces with which they’re familiar, as well as managed services and support.
